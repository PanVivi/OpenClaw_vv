八套当前角色卡相对最近归档总体没有普遍缩减；真实丢失发生在顶层设计文档。被提交 `376bc1f` 删除的 `FinalDesign v1.02` 有 209 行，当前 v1.07 只有 64 行，原有直接联系、架构图、逐角色职责、常驻通信和临时子 Agent 契约被摘要替代。QuickBrief、Workflows、DeploymentPlan 和 ImplementationRoadmap 也出现“新版本用短摘要替代完整基线”的现象。

本轮处理：

1. 从删除前固定 Git 提交精确恢复 `FinalDesign最终设计-v1.02.md`，保持原文不变。
2. 新建 v1.08，以完整 v1.02 为底稿，增量合并后续已验证修正。
3. QuickBrief v1.09、Workflows v0.10、DeploymentPlan v0.10、ImplementationRoadmap v0.11 均以各自早期完整版本为底稿追加现行规则。
4. 当前八套角色卡先原样归档，再升版；共同协议升 v0.05。
5. 不恢复会造成记忆串线的全局 `sessions_history`，不恢复已确认错误的生活路由或 companion 私聊共享。

## 风险分级恢复

| 等级 | 处理 |
| --- | --- |
| 低风险 | 自动执行并验证，不询问少主 |
| 中风险 | 工程链内部 reviewer.Risk、备份、回滚和 Test，不重复询问少主 |
| 高风险 | reviewer.Risk 后由 housekeeper 集中询问一次 |

任务授权、内部 reviewer.Risk 和 OpenClaw 单命令审批分层处理。少主下达目标、对象、范围和完成标准明确的任务后，同范围正常步骤不再逐项索权。Bot Token 是必要输入，不是反复授权理由。
