# 001 OpenClaw架设部署｜DocumentRules 文档编号规则｜v1.03

本文件定义 OpenClaw_vv 项目文档和角色卡的命名、版本与留存方式。

## 一、项目文档

版本化说明文档继续使用：

```text
{English文档名中文文档名}-v{版本}.md
```

新版本建立后旧版本保留，只有薇明确决定时才删除。

## 二、角色卡库稳定目录

角色卡库使用不带版本号的稳定路径：

```text
001-OpenClaw规划设计/AgentCards角色卡/
```

每位常驻 Agent 使用独立目录：

```text
{agent-id}-{正式角色名}/
```

当前目录包括：

```text
housekeeper-賈南風/
ops-魚玄機/
coder-步非煙/
reviewer-合并审查/
life-蕭觀音/
companion-dugu-獨孤伽羅/
companion-wu-武曌/
companion-lv-呂雉/
```

Agent ID 必须与 OpenClaw 实际配置一致。

## 三、当前文件与旧文档

已完成角色卡的 Agent 目录结构：

```text
{agent目录}/
├── README.md
├── IDENTITY.md
├── SOUL.md
├── AGENTS.md
├── USER.md
├── TOOLS.md
├── PERMISSIONS.md
└── 旧文档/
    ├── v1.02/
    └── v1.03/
```

- 当前版本文件直接放在 Agent 目录根部，不在文件名中加入版本号。
- 当前版本号写入 README 和 AGENTS，并由角色卡库 README 索引。
- 新版本同步完成后，旧版完整文件移入 `旧文档/<版本>/`。
- 旧文档不得覆盖、改写或自动删除。
- 同一 Agent 只能有一个当前设计版本和一个当前部署版本标记。
- 旧版草案、部署状态和历史说明一并放入对应版本目录。

## 四、角色包文件

五个 OpenClaw workspace 文件：

```text
IDENTITY.md
SOUL.md
AGENTS.md
USER.md
TOOLS.md
```

`PERMISSIONS.md` 是实际权限配置参考，不直接复制为 OpenClaw 配置。`README.md` 记录版本、状态和验收。

## 五、共同协议

全体 Agent 共同协议保存在：

```text
AgentCards角色卡/共同协议/
```

该目录是仓库权威源。OpenClaw 不会因相对路径引用自动加载任意文件，因此每个 Agent 的 `AGENTS.md` 必须内嵌当前协议的完整执行摘要。只写链接不算接入。

## 六、版本规则

- 新开发角色卡从 `v0.01` 开始。
- 小幅修正递增 `0.01`。
- 已定稿角色卡使用 `v1.0`，后续少量修正使用 `v1.01`、`v1.02`。
- 发生结构性更新时提升阶段版本。
- 已发布但发现问题的版本不得原地改写，应归档并创建新版本。
- 设计版本、部署版本和历史版本必须明确区分。

## 七、迁移规则

旧的 `AgentCards角色卡-v*` 目录完成内容核对和历史同步后，内容移入各 Agent 的 `旧文档/`；旧顶层版本目录不再作为当前入口。迁移时保留原版本、源路径、Git 提交和部署状态，确认无引用遗漏后再删除旧入口。

## 八、禁止事项

- 不再新建带版本号的角色卡库顶层目录。
- 不把多个 Agent 的当前文件混放在同一目录。
- 不以“同上一版”“内容省略”代替完整可部署文件。
- 不让多个版本同时标记为当前版本。
- 不因建立新版本自动删除历史文件。
- 不把角色卡设计状态写成运行环境已部署状态。
