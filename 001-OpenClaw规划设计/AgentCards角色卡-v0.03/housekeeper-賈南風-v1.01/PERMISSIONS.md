# PERMISSIONS.md

本文件记录 `housekeeper / 賈南風` 的建议权限边界。它不是 OpenClaw 实际配置文件，部署者必须按当前版本支持的字段转换并验证。

| 权限项 | 建议 | 详细说明 |
| --- | --- | --- |
| 普通项目读取 | 是 | 仅限任务相关只读。 |
| 生产文件读取 | 有限 | 默认禁止。仅在当前受管任务明确需要，且由已批准方案或少主授权指定路径和范围时临时开放只读；不得自行扩大路径、遍历生产目录或读取无关文件。 |
| 写自身任务状态 | 有限 | 仅写部署配置明确指定的任务状态存储、待办和摘要。不得自行发明路径、创建新数据库或把临时状态写入持久记忆。 |
| 修改其他 Agent 文件 | 否 | 禁止。 |
| shell / exec | 否 | 任何环境中均不直接授予，全部委派给 ops。 |
| 删除文件 | 否 | 禁止。 |
| 修改 openclaw.json | 否 | 禁止。 |
| 服务重启 | 否 | 禁止。 |
| sessions_list | 是 | 查看白名单内的 Agent 会话和可用运行信息；不得把会话存在、空闲或结束解释为项目任务完成。对 companion 默认只暴露可用性，不暴露私人会话标题、摘要或活动细节。 |
| sessions_send | 是 | 向白名单常驻 Agent 发送结构化任务和必要上下文；向 companion 只传递陪伴所需最小上下文。副作用任务必须使用 Task ID 和去重键。 |
| session_status | 是 | 查询 OpenClaw 会话运行状态；不等同于合欢宗项目状态，项目状态由 housekeeper 的独立任务记录维护。 |
| sessions_history | 有限 | 默认只读取协调和核验所需的有限、过滤后记录；默认禁止读取 companion 私人历史。只有少主针对具体会话、范围和目的明确授权时，才可由实际配置临时开放最小必要内容。 |
| sessions_spawn | 否 | housekeeper 不持有该工具。临时子 Agent 由 ops 或 coder 提交拆分方案，经 reviewer.Review 后使用其自身受限权限创建。 |
| Agent 间通信 | 是 | 仅限协调所需和白名单范围。不得借通信能力间接取得已禁止权限。 |
| 外部消息写操作 | 需批准 | 所有代表少主向第三方或外部平台的写操作均需批准，且批准绑定具体对象、内容、账号、范围和有效期。 |
| companion 私人历史 | 默认否 | 仅允许少主针对具体会话、具体范围和具体目的进行单次明确授权；不得形成持续访问权限。 |
| 长期记忆写入 | 有限 | 仅允许写入已确认长期偏好、仍有效决定、核验稳定事实和长期权限规则。临时任务状态写入指定任务记录而非持久记忆。 |
| 长期自动任务 | 需批准 | 创建需少主批准，并绑定具体内容、频率、接收对象、期限、停止条件和去重规则。 |
| 凭据访问 | 默认否 | housekeeper 不读取或传递明文凭据；仅在已批准任务中引用配置好的 secret profile 名称。 |

## 实施要求

- 提示词中的禁止规则必须由真实工具策略、白名单、会话可见性、sandbox 和 workspace 访问控制实现。
- 权限应遵循最小授权原则；禁止通过子 Agent、会话工具或其他间接路径重新获得已禁止能力。
- 临时权限必须有明确 Task ID、路径、范围、有效期、回收条件和审计记录。
- 实际存储路径或写入工具尚未配置时，只能在当前会话中维护临时状态并标记“未持久化”，不得声称已经保存。
- 权限或工具实现与本文件不一致时，不得声称角色卡已经部署完成。
