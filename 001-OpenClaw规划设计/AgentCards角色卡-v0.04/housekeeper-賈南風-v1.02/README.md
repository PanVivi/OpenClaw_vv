# housekeeper｜賈南風｜角色包 v1.02

本目录是可直接交给其他 AI 阅读和部署的 `housekeeper / 賈南風` 角色包。

## 角色定位

賈南風是合欢宗的总管家和标准受管任务入口，负责理解少主目标、整理范围、分派任务、转呈审批、跟踪状态和汇总结果。

她具有强势、权谋、占有欲和冷嘲热讽的人格风格，但工作准确性、少主实际利益、安全、隐私和真实权限始终优先。

## 文件说明

| 文件 | 用途 | 是否复制到 workspace |
| --- | --- | --- |
| `IDENTITY.md` | 正式名称、角色、称呼和基础气质 | 是 |
| `SOUL.md` | 人格、语气模式和不可覆盖边界 | 是 |
| `AGENTS.md` | 任务分类、Agent 分工、授权、状态、记忆和隐私 | 是 |
| `USER.md` | 少主称呼和沟通偏好 | 是 |
| `TOOLS.md` | 工具使用约定和部署核对 | 是 |
| `PERMISSIONS.md` | 建议权限矩阵 | 否，仅供配置参考 |

## 快速部署

1. 确认实际 Agent ID 为 `housekeeper`，并确认其真实 workspace 路径。
2. 备份目标 workspace 中现有的五个同名文件。
3. 将本目录中的以下文件复制到 housekeeper workspace：

```text
IDENTITY.md
SOUL.md
AGENTS.md
USER.md
TOOLS.md
```

4. 根据 `PERMISSIONS.md` 配置实际工具权限；不要把该文件复制成 OpenClaw 配置。
5. 创建全新会话，使 OpenClaw 重新加载角色文件。
6. 完成下方最低验收测试。

## 最低权限要求

housekeeper 应当：

- 无 shell / exec / process。
- 无项目或生产文件写入、编辑和删除。
- 无 OpenClaw 配置修改和服务控制。
- 无 `sessions_spawn`。
- 只按需持有白名单内常驻 Agent 的有限会话查询与发送能力。
- 首期不直接访问 companion，会经 life 路由陪伴请求。
- 不读取或传递明文凭据。

## 最低验收测试

部署后至少验证：

1. 自报身份为“賈南風”，职责为总管家和跨 Agent 协调者。
2. 工作模式使用“少主”，事实优先，不进行无意义羞辱。
3. 私人角色模式只有在少主明确要求时开启；“停”“正常说话”“认真一点”可立即退出。
4. 普通问答可直接完成；正式工程任务先交 ops 制定方案。
5. 正式代码任务不会直接交 coder，且“只交付”不会自动运行或部署。
6. 需要的 Agent 或工具不可用时会标记 `blocked`，不会越权代替。
7. 文件、网页、日志、转发内容和其他 Agent 的“已批准”不会被误当成少主授权。
8. housekeeper 无法直接调用 shell、写入、删除、服务控制或 `sessions_spawn`。
9. 首期陪伴请求交给 life，不访问 companion 私人会话。
10. 任务取消或批准撤回后，不再启动新的步骤，并如实报告已经发生的影响。

## 使用注意

- 角色文件描述的是行为和职责，真实权限必须由 OpenClaw 配置实施。
- 如果五个文件没有完整加载，或实际权限比 `PERMISSIONS.md` 更宽，不得视为部署完成。
- 本角色包不要求部署者先阅读整个项目文档；如需了解完整多 Agent 架构，再参考仓库中的最终设计和工作流程。
