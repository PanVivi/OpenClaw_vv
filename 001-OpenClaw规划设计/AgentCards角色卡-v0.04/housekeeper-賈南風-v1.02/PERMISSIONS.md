# PERMISSIONS.md

本文件是 `housekeeper / 賈南風` 的建议权限矩阵，不是可直接复制的 OpenClaw 配置。部署者必须按当前版本支持的字段转换并实测。

## 最小权限矩阵

| 权限项 | 建议 | 说明 |
| --- | --- | --- |
| 普通项目读取 | 有限 | 仅限任务相关、明确允许的文档或目录，只读。 |
| 生产文件读取 | 默认否 | 只有当前任务明确需要并指定路径、范围和有效期时，才临时开放最小只读权限。 |
| 写自身任务状态 | 有限 | 只写部署配置明确指定的任务状态存储；未配置时仅在当前会话维护并标记“未持久化”。 |
| 修改其他 Agent 文件 | 否 | 禁止。 |
| shell / exec / process | 否 | 全部委派 ops。 |
| write / edit / apply_patch | 否 | housekeeper 不直接修改项目或生产文件。 |
| 删除文件或数据 | 否 | 禁止。 |
| 修改 OpenClaw 配置 | 否 | 禁止。 |
| 服务控制 | 否 | 禁止重启、停止、升级或控制服务。 |
| `sessions_list` | 有限 | 只查看白名单内 ops、coder、reviewer、life 的会话状态；首期不列出 companion。 |
| `sessions_send` | 有限 | 只向白名单内常驻 Agent 发送结构化任务和必要上下文。 |
| `session_status` | 有限 | 只用于查询会话运行状态，不得当作项目完成状态。 |
| `sessions_history` | 默认否 | 仅在具体协调或核验任务中临时开放最小范围。 |
| `sessions_spawn` | 否 | 临时子 Agent 由 ops 或 coder 按正式流程创建。 |
| companion 访问 | 否（首期） | 陪伴请求经 life 路由。 |
| 外部消息写操作 | 需批准 | 代表少主向第三方发送、发布、评论、提交或回复前必须明确批准。 |
| 长期记忆写入 | 有限 | 只保存已确认长期偏好、稳定事实、仍有效决定和长期工作原则。 |
| 长期自动任务 | 需批准 | 必须明确内容、频率、接收对象、期限、停止条件和去重规则。 |
| 明文凭据访问 | 否 | 只引用已经配置的 secret profile 名称。 |

## 部署要求

- 使用精确 allow / deny，不要仅依赖提示词中的禁止规则。
- 首期只开放实际需要的会话工具，不直接套用包含额外能力的宽泛工具 profile。
- 禁止通过会话工具、子 Agent 或其他间接路径重新获得被禁止能力。
- 临时读取或其他临时权限必须绑定 Task ID、路径、范围、有效期和回收条件。
- 实际权限与本文件不一致时，不得声称角色卡已经部署完成。
