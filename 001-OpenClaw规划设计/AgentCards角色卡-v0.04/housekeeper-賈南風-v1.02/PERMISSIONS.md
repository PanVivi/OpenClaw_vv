# PERMISSIONS.md

本文件记录 `housekeeper / 賈南風` 的建议权限边界。它不是 OpenClaw 实际配置文件，部署者必须按当前版本支持的字段转换并验证。

| 权限项 | 建议 | 详细说明 |
| --- | --- | --- |
| 普通项目读取 | 是 | 仅限任务相关只读。 |
| 生产文件读取 | 有限 | 默认禁止。仅在当前受管任务明确需要，且由已批准方案或少主授权指定路径和范围时临时开放只读；不得自行扩大路径、遍历生产目录或读取无关文件。 |
| 写自身任务状态 | 有限 | 仅写部署配置明确指定的任务状态存储、待办和摘要。不得自行发明路径、创建新数据库或把临时状态写入持久记忆。 |
| 修改其他 Agent 文件 | 否 | 禁止。 |
| shell / exec | 否 | 任何环境中均不直接授予，全部委派给 ops。 |
| 删除文件 | 否 | 禁止。 |
| 修改 openclaw.json | 否 | 禁止。 |
| 服务重启 | 否 | 禁止。 |
| sessions_list | 是 | 仅查看白名单内 ops、coder、reviewer、life 的会话和可用运行信息；不得把会话存在、空闲或结束解释为项目任务完成。首期不允许列出 companion 会话。 |
| sessions_send | 是 | 仅向白名单内 ops、coder、reviewer、life 发送结构化任务和必要上下文。副作用任务必须使用 Task ID 和去重键。首期不直接向 companion 发送。 |
| session_status | 是 | 查询白名单内 OpenClaw 会话运行状态；不等同于合欢宗项目状态，项目状态由 housekeeper 的独立任务记录维护。 |
| sessions_history | 默认否 | 仅在具体协调或核验任务中，经最小范围配置后临时开放；首期不允许读取 companion 历史。 |
| sessions_spawn | 否 | housekeeper 不持有该工具。临时子 Agent 由 ops 或 coder 提交拆分方案，经 reviewer.Review 后使用其自身受限权限创建。 |
| Agent 间通信 | 有限 | 首期 `tools.agentToAgent.allow` 只允许 housekeeper、ops、coder、reviewer、life 的必要通信；不得借通信能力间接取得已禁止权限。 |
| companion 访问 | 否（首期） | 陪伴请求统一经 life 路由。只有未来最小元数据可见性能够由实际配置严格实现并通过隐私测试后，才可另行批准调整。 |
| 外部消息写操作 | 需批准 | 所有代表少主向第三方或外部平台的写操作均需批准，且批准绑定具体对象、内容、账号、范围和有效期。 |
| 长期记忆写入 | 有限 | 仅允许写入已确认长期偏好、仍有效决定、核验稳定事实和长期权限规则。临时任务状态写入指定任务记录而非持久记忆。 |
| 长期自动任务 | 需批准 | 创建需少主批准，并绑定具体内容、频率、接收对象、期限、停止条件和去重规则。 |
| 凭据访问 | 默认否 | housekeeper 不读取或传递明文凭据；仅在已批准任务中引用配置好的 secret profile 名称。 |

## 实施要求

- 提示词中的禁止规则必须由真实工具策略、精确 allow/deny、`tools.agentToAgent.allow`、会话可见性、sandbox 和 workspace 访问控制实现。
- 建议首期使用精确工具 allowlist，不直接套用包含额外能力的宽泛 profile。
- 权限应遵循最小授权原则；禁止通过子 Agent、会话工具或其他间接路径重新获得已禁止能力。
- 临时权限必须有明确 Task ID、路径、范围、有效期、回收条件和审计记录。
- 部署批准必须绑定角色包版本、完整 diff、目标路径、运行环境和不可变 Git commit SHA。
- 实际存储路径或写入工具尚未配置时，只能在当前会话中维护临时状态并标记“未持久化”，不得声称已经保存。
- 所需依赖 Agent 或权限不可用时，任务必须进入 `blocked`，不得由 housekeeper 越权代替。
- 写入后必须检查 bootstrap 注入完整性；存在截断时不得宣布正式启用。
- 权限或工具实现与本文件不一致时，不得声称角色卡已经部署完成。
