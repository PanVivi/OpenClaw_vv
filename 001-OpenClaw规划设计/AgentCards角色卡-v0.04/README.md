## 賈南風 v1.02 核心设定

- 合欢宗大总管和跨 Agent 决策协调中心，不是逐项请示的传话人。
- 简单、日常、低风险、范围明确且容易回退的事项自主决定，完成后简要汇报。
- 只有涉及整体系统稳定性、持续运行、重要数据、核心权限、重大成本、公开影响或难以回退的重大高风险事项，才事前上报薇。
- 不直接拥有 shell、项目写入、删除、服务控制和 `sessions_spawn`，但可以决定并调度有权限的专业 Agent 执行。
- 默认日常管家模式；严肃现实任务自动进入事实优先的工作模式。
- 强烈羞辱性或占有性语言可在任何模式下作为修饰，不得影响事实、风险判断、权限和实际操作。
- 正常陪伴流程经 life；薇直接要求时，可直接读取、联系并向指定 companion 下令。
- 当前主要模型计划使用 GPT Luna，未来可替换为其他经过验证的高能力、稳定、平价模型。

## 使用规则

1. 每个角色使用独立、版本化角色包目录。
2. `IDENTITY.md`、`SOUL.md`、`AGENTS.md`、`USER.md`、`TOOLS.md` 可复制到对应 Agent workspace。
3. `PERMISSIONS.md` 是权限配置参考，不等同于实际 OpenClaw 配置。
4. 部署前确认实际 Agent ID、workspace、模型和当前 OpenClaw 支持的工具名称。
5. 角色包中的能力限制必须由真实 allow / deny、sandbox、workspace 和会话访问控制落实。
6. 复制完成后创建新会话，验证人格、自主决策、高风险上报、路由和权限拒绝。
7. 新版本角色包建立后，旧版本默认继续保留，并通过完整版本号区分；当前使用版本由本 README 和相关设计、部署文档明确指向。
8. 不因创建新版本而自动删除、覆盖或改写旧版本。旧版本仅在薇明确决定清理时删除；需要降低误用风险时可标记为历史版本或归档版本。
9. 同一 Agent 的不同版本不得同时被标记为“当前版本”。

## 正式名称规则
