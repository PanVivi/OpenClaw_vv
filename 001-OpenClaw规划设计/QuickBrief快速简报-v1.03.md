## 项目目标

为薇搭建长期运行的 OpenClaw 多 Agent 系统。组织名为合欢宗，支持 Telegram Bot、任务调度、工程部署、内部审查、生活管理和独立陪伴分支。

## 常驻 Agent

```text
housekeeper       賈南風：大总管、决策与协调中心
ops               魚玄機：工程方案、调查、调试和执行
coder             步非煙：代码与脚本产出
reviewer          合并审查：Review / Risk / Test
life              蕭觀音：生活总管、日程提醒、三位 companion 日常管理
companion-dugu    獨孤伽羅：独立陪伴 Agent
companion-wu      武曌：独立陪伴 Agent
companion-lv      呂雉：独立陪伴 Agent
```

三位 companion 各自拥有独立会话和独立 Telegram Bot，可以同时运行，只负责聊天、陪伴和情绪价值，不具有工程执行权限。

## 角色卡入口

```text
001-OpenClaw规划设计/AgentCards角色卡/
```

每位 Agent 有独立目录。当前文件直接显示在 Agent 目录，历史版本保存到该目录的 `旧文档/`。

当前设计版本：

- 蕭觀音 v0.02：待部署验收。

## 賈南風

賈南風是自主决策的大总管。低风险、范围明确、容易回退的事项自行推进；重大系统稳定性、重要数据、核心权限、重大成本、公开影响或难以回退事项才事前上报薇。她不直接持有 shell、普通文件写入、删除、服务控制或 `sessions_spawn`，需要时调度专业 Agent。

## 蕭觀音

蕭觀音负责全部生活、娱乐、天气、出行、一般健康习惯、日历、主动提醒和三位 companion 日常管理。每日当地时间 06:00 通过指定 Telegram Bot 发送自然口语化的晨间消息，不写成报告。

## 共同协议

Agent 可以在职责所需范围共享少主明确表达的偏好、当前状态和近期相关信息，必须标明来源、可信程度、适用范围和失效条件。宫斗、争宠和暗讽只能影响表达，不得影响真实任务、权限、证据和验收。

## 当前运行限制

已部署的賈南風 v1.02 角色主体正常，但跨 Agent 发送仍受全局 A2A 策略限制，跨 Agent 历史读取仍受 session visibility 限制。新角色卡部署前必须单独审查并落实真实会话、记忆、Telegram、天气、日历和 Cron 权限。
