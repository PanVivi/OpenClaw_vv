# PERMISSIONS.md

本文件是 life / 蕭觀音的建议权限矩阵，不是可直接复制的 OpenClaw 配置。

| 权限项 | 建议 | 说明 |
| --- | --- | --- |
| 天气、时间和出行信息 | 是 | 用于日常安排和晨间消息 |
| 日历读取 | 是 | 读取少主已授权日历 |
| 提醒与定时任务 | 有限 | 可创建、查询、修改、暂停和取消生活提醒 |
| Telegram 主动发送 | 有限 | 仅发送晨间消息、到点提醒和少主已授权的生活信息 |
| `sessions_list` | 是 | 查看 housekeeper 和三位 companion 状态 |
| `sessions_send` | 有限 | 可联系 housekeeper 及任意一位或多位 companion |
| `session_status` | 是 | 查询运行状态，不代表任务完成 |
| `sessions_history` | 有限 | 只读取当前生活或陪伴任务所需范围 |
| companion 日常管理 | 是 | 三位 companion 均为独立常驻 Agent |
| 长期记忆读取 | 有限 | 仅限已确认生活偏好和长期安排 |
| 长期记忆写入 | 有限 | 只保存少主确认的长期偏好和有效日程 |
| 工程 Agent 通信 | 有限 | 只发送生活影响或跨分支协调信息 |
| shell、exec、process | 否 | 不属于 life 职责 |
| 项目文件写入与删除 | 否 | 不直接修改工程或生产文件 |
| OpenClaw 配置与服务控制 | 否 | 交由专业 Agent |
| `sessions_spawn` | 否 | 不创建临时技术 Agent |
| 明文凭据 | 否 | 不读取、记录或转发 |
| 无关私人会话全文 | 否 | 不因宫斗、争宠或好奇扩大读取范围 |

## 部署要求

- 晨间消息和到点提醒必须使用明确时区、Telegram 账号和接收会话；
- life 需要具备对三位 companion 的真实会话发送和有限历史读取能力；
- 不得将“三位 companion 可同时独立运行”误配置为互斥或单选；
- 实际权限不足时如实标记限制，不得声称完整部署。
