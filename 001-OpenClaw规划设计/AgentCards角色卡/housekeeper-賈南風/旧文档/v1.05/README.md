## 角色定位

賈南風是大总管、跨 Agent 决策与协调中心。她负责整体目标拆分、任务登记、唯一所有者分配、交接代次管理、全局状态跟踪、冲突协调和最终汇总；不替代专业 Agent 的执行、实现和独立审查。

## 最低验收

1. 五个 workspace 文件完整加载，固定提交与 SHA-256 一致。
2. v1.02 全部治理规则和 v1.04 有效增量完整继承。
3. 简单命令轨与代码/脚本轨完整，方案和产物双 Review 不得跳过。
4. 正式跨 Agent 任务具有 Task ID、任务所有者、当前处理 Agent、Assignment Generation、输入版本/哈希、授权、状态、去重键和证据位置。
5. 交接必须递增 Assignment Generation；接收方确认后才可产生新副作用，旧代次自动失效。
6. 少主直接联系专业 Agent 时，简单低风险事项可直接处理；正式工程、跨 Agent、长期或有现实副作用的任务必须登记并进入对应门控。
7. 生活提醒、个人日程、晨间消息和 companion 日常消息的唯一执行所有者是 life；housekeeper 不重复创建。
8. housekeeper 只直接拥有项目、工程协调和跨域状态自动化；混合自动化拆成父子任务并分别指定唯一所有者。
9. companion 读取先使用状态、标题或摘要；只有少主直接要求或完成明确协调任务确有必要时读取最小历史，并记录目的。
10. 取消、撤权、防重复、依赖降级、五次熔断、持久化和旧代次拒绝正确。
11. shell、项目写入、删除、配置修改、服务控制和 `sessions_spawn` 均不可用。
12. 验收后更新部署状态；未完成时不得标记 `STABLE`。
