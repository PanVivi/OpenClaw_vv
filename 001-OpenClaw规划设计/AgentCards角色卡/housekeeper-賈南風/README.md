## v1.04 增量新增

- 内嵌共同协议 v0.02 的执行规则。
- 明确三位 companion 可同时独立运行。
- life / 蕭觀音负责生活分支、companion 日常管理、晨间消息和生活提醒。

以上新增不得替代或削弱 v1.02 的工程双审、简单命令轨、代码/脚本轨、任务状态机、取消撤权、防重复、临时技术子 Agent、外部消息、凭据和真实性规则。

## 最低验收

1. 五个 workspace 文件完整加载，无缺失、无截断。
2. v1.02 全部治理规则逐条继承，无静默删减。
3. 简单命令轨与代码/脚本轨分离；代码实现后必须经过 ops 核对和 reviewer.Review 产物审查。
4. 低风险自主决策和重大高风险上报规则正确。
5. 防重复、完整任务状态、取消撤权和五次熔断正确。
6. 技术子 Agent 的字段、隔离上下文、权限和 reviewer 门控正确。
7. 外部消息、凭据和工具真实性规则在运行时注入文件中可见。
8. A2A 真实发送与受限跨 Agent 历史读取可用。
9. 三位 companion 可同时独立运行，日常管理经 life；只有少主直接要求时，housekeeper 才可直达指定 companion。
10. 共同协议信息包含来源、可信程度、适用范围和失效条件。
11. shell、普通文件写入、删除、配置修改、服务控制和 `sessions_spawn` 均不可用。
12. 验收完成后立即更新根目录部署状态；权限不完整时只能记录为 `partially completed`、`blocked` 或 `not verified`。
