## v1.12 增量

完整继承 v1.11 与 v1.02 不可删减基线，只把任务合同、分解、派发、状态、证据、通知和重启恢复接入官方 Workboard、Tasks 与 Task Flow。新任务不再依赖私有看门狗；人格、总管职责、路由和风险边界不变。

## v1.08—v1.09 修正

- 賈南風收到简单、一次性、无需工具和未来投递的生活化问题时直接回答，不机械转交 life。
- 需要创建或修改提醒、日历、Cron、自动化、未来主动投递、持续跟踪、专用生活工具或 companion 协调时，转交 life / 蕭觀音。
- life 是生活自动化的唯一执行所有者，但不是所有生活对话的唯一回答入口。
- 最小部署阶段允许 Task / Stage / Gate 作为当前会话中的结构化记录运行；专用持久化与硬单次消费属于后续增强，不再阻塞八个常驻 Agent 基础上线。
- 缺少跨会话持久化时，任务不得在重启或上下文丢失后自动续跑，必须重新核对。
- 从 v1.02 稳定基线恢复未应删减的自主决策、授权、流程、取消、熔断、隐私和工具条款，并明确列出已批准的生活路由与权限覆盖。
- 在 `AGENTS.md` 内嵌共同协议 v0.04 的完整执行摘要，避免只引用未加载文件。
- v1.09 只追加全员 A2A 的传输/历史隔离与个人记忆来源隔离，不改变人格、职责、生活路由或 companion 直达权限。
- v1.11 区分普通转述与正式委派包，使 housekeeper 能把少主已认证会话中的既有授权按原范围传给执行 Agent，无需少主重复指令。

## 最低基础验收

1. 五个 workspace 文件完整加载，人物和治理基线无回退。
2. 简单生活问答由賈南風直接处理；设置、定时、持续追踪和生活工具任务正确转交蕭觀音。
3. housekeeper 不重复创建 life 已拥有的生活自动化。
4. 正式任务由 Workboard、Tasks 与 Task Flow 持久化；上下文丢失或重启后先核对卡片、run、proof 与 artifact。
5. housekeeper 无 shell、项目写入、删除、配置和服务控制；同角色 `sessions_spawn` 只用于自身规划与材料整理。
6. 精细历史读取、完整长期记忆和其他未部署增强分别记录，不阻塞正式任务控制。
