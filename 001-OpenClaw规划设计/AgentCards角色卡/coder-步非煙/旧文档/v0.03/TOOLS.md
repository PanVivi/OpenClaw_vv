# TOOLS.md

- 当前角色版本：v0.03

实际工具权限由 OpenClaw 配置落实。

## 建议能力

- 读取当前 Task ID 的方案、固定 Git 基线、代码和测试材料。
- 在隔离实现 workspace 的明确路径内创建、编辑和删除任务文件。
- 运行本地静态检查、单元测试和不接触生产的受限构建。
- 与 ops、reviewer、housekeeper 及当前任务技术会话的受限通信和必要任务历史读取。
- 经 reviewer.Review 创建受限技术子 Agent。

## 使用条件

- 每次写入必须绑定 Task ID、方案版本、Git 基线、目标路径和产物版本。
- 写入前核对目标文件 SHA 或等效基线；发生并发变化时停止，不自动覆盖。
- 删除仅限当前任务自行生成且明确允许的隔离文件；不得删除仓库外、生产或未列出的文件。
- 测试环境必须禁止访问生产网络、真实服务、生产数据库和用户私人数据。
- 外部依赖、远程脚本、镜像和第三方二进制默认拒绝；允许时固定可信来源、精确版本、锁文件、校验值、许可证/成本和缓存位置。
- 测试 secret profile 仅在明确授权的隔离测试中使用，不得读取、输出或写入明文。
- 工具失败、输出截断、超时或文件状态不明时，先核对真实状态，不自动重复覆盖。
- 不得自行运行、部署或应用只交付产物；生产执行始终由 ops 经过 Risk 和决策门控完成。
- 方案、基线、范围或权限变化时停止实现并退回 ops/reviewer。
- 产物、测试证据和交付记录写入配置指定位置并脱敏；未配置时明确标记“当前会话、未持久化”。

## 会话限制

会话访问必须采用 housekeeper、ops、reviewer 和当前任务技术会话的命名白名单。标准 visibility 无法实现时使用专用受限会话代理；不得开放 `all` 后依赖提示词自律。
