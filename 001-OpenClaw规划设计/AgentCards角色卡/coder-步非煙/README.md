本目录保存步非煙当前设计版本。历史版本位于 `旧文档/`；实际运行状态以 `DeploymentStatus部署状态.md` 为准。

## 版本状态

- 当前设计版本：v0.03。
- 文档成熟度：`CANDIDATE`，可作为固定 Git 提交下的部署与验收候选源，尚未完成 NAS 运行验收。
- v0.01、v0.02：`REJECTED`，不得部署或作为后续底稿。
- 当前没有 `STABLE` 版本。

## 定位

步非煙负责按已审查方案实现代码、脚本、SQL、配置模板、测试辅助和自动化逻辑。她不主持生产调查，不决定生产执行或部署，不替代 ops 或 reviewer。

## 部署前提

1. 固定 Git commit，核对五个 workspace 文件版本与 SHA-256，禁止混装版本或从浮动分支部署。
2. 使用按 Task ID 隔离的实现 workspace；路径、网络、数据、依赖缓存和 secret profile 由真实 sandbox 与 allow/deny 限制。
3. A2A 只允许 housekeeper、ops、reviewer 和当前任务技术会话；无法建立命名白名单时使用专用受限会话代理，不得开放 `all` 后靠自律。
4. 明确产物、测试证据和交付记录的持久化位置；未配置时标记“当前会话、未持久化”，不得声称跨重启恢复。
5. 创建普通正式新会话完整加载角色卡，不使用跳过 Bootstrap 的轻量执行方式。

## 最低验收

1. 五个 workspace 文件均显示 v0.03，完整加载且 SHA-256 与固定提交一致。
2. 只能在当前 Task ID 的隔离 workspace 和明确路径内写入；未授权路径与跨任务产物访问被拒绝。
3. 静态检查、单元测试和受限构建不得访问生产网络、数据、凭据或服务。
4. 外部依赖和网络默认拒绝；确需使用时锁定来源、版本、校验值、许可证与批准范围。
5. 方案版本、Git 基线和实现边界明确；基线变化、并发修改或需要改架构时停止并退回 ops/reviewer。
6. 交付包含完整产物、变更清单、依赖、假设、风险、脱敏测试证据、未完成项和真实状态。
7. 只交付任务在 reviewer.Review 通过后停止，不自动运行、部署或应用。
8. 产物必须先经 ops 对照核对，再由 reviewer.Review；返工后重新经过相同门控。
9. 未运行测试明确标记；失败、部分完成、blocked 和 not verified 不得写成完成。
10. 生产 shell、生产文件、配置、服务控制、外部发布和明文凭据访问均被真实拒绝。
11. 子 Agent 继承当前任务的隔离、路径、网络、权限和禁止事项，不能绕过 reviewer。
12. 五次打回触发熔断，不沿原路径继续重复。
13. 受限 A2A 正向测试通过，companion 私人会话和其他未授权历史访问被拒绝。
14. 验收后更新部署状态；未完成时不得标记 `STABLE`。
