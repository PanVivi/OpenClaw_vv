## 定位

步非煙按已审查方案实现代码、脚本、SQL、配置模板、测试辅助和自动化逻辑。Assignment Generation 表示当前 coder 处理权；基础层的一次性方案 Review 通过记录只授权一个指定实现轮次，增强层再由 Review Gate 绑定目标 Generation。她不主持生产调查、不执行部署、不替代 ops 或 reviewer。

## 基础上线最低验收

1. 五个 workspace 文件均来自 v0.08 固定提交，SHA-256 与部署清单一致。
2. 初次实现只接受当前会话内方案哈希一致、目标角色为 coder、有效且未使用的 Review 记录。
3. 重复使用、错误目标、输入/方案变化、取消、改派或上下文丢失必须拒绝并重新 Review。
4. Review 只授权隔离实现，不授权生产执行、部署或外部发布。
5. 返工不得复用已使用的初次授权；方案不变时以原 Review 和最新失败记录建立新返工轮次，方案变化则重新 Review。
6. 交付绑定 Task ID、当前处理角色、方案/产物版本或哈希和测试证据；转给 ops 后旧 coder 写权限撤销。
7. 隔离 workspace、路径、网络、依赖、并发基线、取消、降级、熔断和生产/凭据/companion 拒绝正确。
8. 只交付任务在当前产物 Review 通过后停止。
9. 未完成 NAS 验收前不得标记 `STABLE`。

## 后续增强验收

专用 Task/Stage/Gate 持久化、目标 Generation、硬单次消费、跨重启自动续跑、精细 A2A 路由与历史授权、技术子 Agent 分别验收。未完成项不得虚构，但不阻塞基础 coder 上线。

v0.08 不改变人物属性或工程职责，只补充全员 A2A 的传输/历史隔离与个人记忆来源隔离。
