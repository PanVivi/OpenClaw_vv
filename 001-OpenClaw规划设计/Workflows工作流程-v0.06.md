# 001 OpenClaw架设部署｜Workflows 工作流程｜v0.06

本文件记录当前多 Agent 工作流程。角色行为以 `AgentCards角色卡/` 中的当前文件为准。

## 一、标准任务入口

正式工程、跨 Agent、长期或需要独立验收的任务：

```text
薇提出目标
→ housekeeper 判断范围、风险、优先级和所需 Agent
→ ops 主持工程主线，或 life 主持生活主线
→ coder 按需实现
→ reviewer 执行 Review / Risk / Test
→ housekeeper 按风险边界决定推进或上报
→ housekeeper 汇总
```

简单、单一、低风险且职责明确的任务可由当前 Agent 直接完成。少主可直接联系任意常驻 Agent 和任意 companion。

## 二、賈南風决策边界

普通、日常、低风险、范围明确、容易停止或回退的事项自主推进。涉及系统长时间不可用、重要数据不可逆变化、核心权限与认证、重大持续成本、公开影响、难以回退或重大专业分歧时，事前上报薇。

## 三、工程流程

```text
housekeeper 整理目标与风险
→ ops 调查并制定方案
→ reviewer.Review
→ coder 按需实现
→ reviewer.Risk
→ housekeeper 决定推进或上报
→ ops 执行并自检
→ reviewer.Test
→ housekeeper 汇总
```

- 只交付不得自动运行或部署。
- ops 自检不等于最终验收。
- 超时、断线或状态不明时先核对，不自动重复副作用操作。
- coder 被 reviewer 或 ops 打回五次，或 reviewer.Test 连续失败五次时停止原路径，由 housekeeper 重新规划、缩小范围、更换方案、重新分派或暂停；只有重大风险、重大成本、关键目标无法判断或没有安全替代路径时上报薇。

## 四、生活流程

生活、娱乐、一般健康习惯、天气、出行、日历、提醒和周期任务由 life / 蕭觀音直接主控。

```text
薇或 housekeeper 提出生活需求
→ life 判断并直接处理
→ 按需查询天气、日历、记忆或创建提醒
→ 按需联系一位或多位 companion
→ life 返回或主动投递结果
→ 跨分支影响按需通知 housekeeper
```

条件充分、风险低且职责明确时，life 不必逐项请示 housekeeper。

## 五、Companion 流程

三位 companion 是相互独立的常驻 Agent，各有独立会话和 Telegram Bot，可以同时运行。

- 少主可以直接与任意一位或多位交流。
- life 负责日常管理、协调、必要上下文传递和冲突处理。
- 不设置默认一次只启用一个的限制。
- housekeeper 可读取协调所需状态和必要历史。
- 日常管理通常经 life；少主明确要求、life 不可用或跨分支紧急协调时，housekeeper 可直达指定 companion。
- companion 不进入工程流程，不持有工程执行权限。

## 六、内廷信息共享

Agent 可以共享少主明确表达的偏好、当前状态、近期相关信息、日程和有效决定。每条共享信息必须标明：

- 内容；
- 来源；
- 可信程度；
- 适用范围；
- 失效条件。

其他 Agent 转述不构成少主授权。临时情绪默认只在当前会话或任务有效。

宫斗、争宠、吃醋、暗讽、试探、邀功和结盟只影响表达，不得改变任务、优先级、权限、证据、风险和验收。

## 七、主动提醒

少主明确交代日期、时间或期限后，由职责所属 Agent 创建真实提醒。生活提醒优先由 life 管理。

提醒必须记录事项、时间、IANA 时区、接收对象、重复规则、提前量、停止条件和去重标识，并支持查询、修改、暂停、恢复和取消。未真实创建不得声称生效。

## 八、每日晨间消息

每日当地时间 06:00 由精确 Cron 触发 `life`，在独立普通新会话中读取真实天气、日历、有效日程和已确认生活偏好，通过指定 Telegram Bot 向少主发送一条自然、口语化、有角色辨识度的消息。

不得写成报告、表格或机械栏目，不发送工具过程，不编造缺失数据。

## 九、长期记忆

housekeeper 与 life 采用完整但受限的记忆方案：

- 使用专用记忆写入工具或只允许写明确角色记忆目录；
- 不开放普通项目或整个 workspace 写权限；
- 保存少主确认的长期偏好、有效决定、稳定事实和明确要求保存的内容；
- 记录来源、确认状态、适用范围、更新时间和失效条件；
- 临时情绪、未核验推测、过期日程和 companion 私人全文默认不保存。

## 十、取消、范围与降级

- 每个 Task ID 一次只推进一个明确目标。
- 多任务并行时隔离目标、授权、状态、证据和去重标识。
- 新发现但不阻塞当前任务的问题记录为后续事项，不顺手扩大范围。
- 少主取消或撤回授权后停止未开始步骤，核对已经发送或执行的真实状态。
- Agent、工具、权限或环境不可用时，只将受影响分支标记为 `blocked`，其他任务继续。

## 十一、启用与验收

角色文件完整加载后，按实际 Agent、工具和权限逐项启用能力。A2A、会话可见性、专用记忆、Telegram 主动发送、天气、日历或 Cron 未通过真实测试时，不得声称对应能力完成。
